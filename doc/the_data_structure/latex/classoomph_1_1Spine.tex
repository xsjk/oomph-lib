\doxysubsection{Detailed Description}
Spines are used for algebraic node update operations in free-\/surface fluid problems\+: They form the back-\/bones along which nodes in a a free-\/surface mesh are located. Typically, the free surface is located at the "{}end"{} of the spine; the nodes in the interior of the mesh are located at fixed fractions along the spine. The key \doxylink{classoomph_1_1Data}{Data} member of the \doxylink{classoomph_1_1Spine}{Spine} object is its "{}height"{} -- usually an unknown in the problem -- which is used by the \doxylink{classoomph_1_1SpineNode}{Spine\+Node}\textquotesingle{}s node update function to update the \doxylink{classoomph_1_1SpineNode}{Spine\+Node}\textquotesingle{}s position. 

In more complex problems (such as the case where a fluid layer is deposited on an elastic body), the node update function can depend on additional information, such as the \doxylink{classoomph_1_1GeomObject}{Geom\+Object} representation of the elastic body, and additional \doxylink{classoomph_1_1Data}{Data} objects wich specify the position on the \doxylink{classoomph_1_1GeomObject}{Geom\+Object} from which the \doxylink{classoomph_1_1Spine}{Spine} emanates. The \doxylink{classoomph_1_1Spine}{Spine} class therefore provides storage for pointers to Geom\+Objects and storage for any additional geometric \doxylink{classoomph_1_1Data}{Data} that may be required during node update operations.
