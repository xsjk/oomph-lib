3\mainpage Time-harmonic acoustic fluid-structure interaction problems on unstructured meshes
4
5
6In this document we discuss the solution of
7time-harmonic acoustic fluid-structure interaction problems
8on unstructured meshes.
9
10 The driver code is very similar to the one presented in
11<a href=../../acoustic_fsi_annulus/html/index.html>another tutorial</a>
12and we only discuss the changes necessary to deal with the generation
13of the unstructured mesh for the solid domain and the
14assignment of different material properties to different parts of the domain.
15

19\section test A test problem
20
21The sketch below shows the problem setup: A 2D elastic annulus
22which is reinforced with two T-ribs is immersed in a compressible
23fluid and subjected to a time-periodic pressure load of magnitude
24\f[
25{\bf t} = P ( \exp(\alpha(\varphi-\pi/4)^2) + \exp(\alpha(\varphi-3\pi/4)^2) )
26\f]
27(where \f$ \varphi \f$ is the polar angle) along
28its inner surface. The parameter \f$ \alpha \f$ controls the
29"sharpness" of the pressure load. For \f$ \alpha=0 \f$ we obtain a
30uniform, axisymmetric load; the sketch below shows the pressure
31distribution (red vectors indicating the traction) for \f$ \alpha = 200. \f$
32
33\image html setup.gif "Sketch of the problem setup. "
34\image latex setup.eps "Sketch of the problem setup. " width=0.8\textwidth
35
36The structure is symmetric and we only discretise the right half
37(\f$ x_1 > 0 \f$) of the domain and apply symmetry conditions
38(zero horizontal displacement) on the \f$ x_2-\f$ axis.

55
56Here is a plot of the corresponding fluid displacement potential, a
57measure of the fluid pressure:
58
59\image html potential_unstr.gif "The fluid displacement potential, a measure of the fluid pressure. Elevation: real part; contours: imaginary part. "
60\image latex potential_unstr.eps "The fluid displacement potential, a measure of the fluid pressure. Elevation: real part; contours: imaginary part. " width=0.8\textwidth
61
62This looks very pretty and shows that we can solve acoustic FSI
63problems in non-trivial geometries but should you believe the results?
64Here's an attempt to convince you: If we make the rib much softer than the
65annulus and set its inertia to zero the rib will not offer much
66structural resistance
67and the annular region will deform as if the rib was not present.
68If we then set \f$ \alpha = 0 \f$ we apply an axisymmetric forcing
69onto the structure and would expect the resulting displacement
70field (at least in the annular region) to be axisymmetric.
71
72The animation of the displacement field for this case, shown below,
73shows that this is indeed the case:
74

88\section num The numerical solution
89The driver code for this problem is very similar to the one discussed in
90<a href=../../acoustic_fsi_annulus/html/index.html>another tutorial</a>.
91Running \c sdiff on the two driver codes
92<CENTER>
93<A HREF="../../../../demo_drivers/interaction/acoustic_fsi/acoustic_fsi.cc">
94demo_drivers/interaction/acoustic_fsi/acoustic_fsi.cc
95</A>
96</CENTER>
97and
98<CENTER>
99<A HREF="../../../../demo_drivers/interaction/acoustic_fsi/unstructured_acoustic_fsi.cc">
100demo_drivers/interaction/acoustic_fsi/unstructured_acoustic_fsi.cc
101</A>
102</CENTER>
103shows you the differences, the most important of which are:
104- The use of <a href="../../../helmholtz/scattering/html/index.html#ABCs">
105 approximate/absorbing boundary conditions (ABCs)</a>
106 rather than a <a href="../../../helmholtz/scattering/html/index.html#DtN">
107 Dirichlet-to-Neumann mapping</a> for the Helmholtz equation,
108 because the boundary along which the Sommerfeld radiation condition
109 is applied is not a full circle.
110 \n\n
111- The provision of multiple elasticity tensors and frequency
112 parameters for the two different regions (the rib and the
113 annulus).
114 \n\n
115- The change of forcing from a prescribed time-harmonic displacement
116 to a pressure
117 load on the inside boundary -- this requires yet another mesh of
118 \c FaceElements.
119 \n\n
120- The provision of a helper function \c complete_problem_setup()
121 which rebuilds the elements (by passing the problem parameters
122 to the elements) following the unstructured mesh adaptation.
123 (The need/rationale for such a function is discussed in
124 <a href="../../../meshes/mesh_from_inline_triangle/html/index.html">
125 another tutorial.</a>)
126 \n\n
127- The mesh generation -- the specification of the curvilinear
128 boundaries and the geometry of the rib is somewhat tedious.
129 We refer to
130 <a href="../../../meshes/mesh_from_inline_triangle_internal_boundaries/html/index.html">
131 another tutorial</a> for a discussion of how to define the
132 internal mesh boundary that separates the two regions
133 (the rib and the annular region) so that we can assign
134 different material properties to them.
135 \n\n
136.
137All of this is reasonably straightforward and provides a
138powerful code that automatically adapts both meshes while
139respecting the curvilinear boundaries of the domain.
140Have a look through the driver code and play with it.
