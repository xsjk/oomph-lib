11\section setup Setting up the documentation
12
13\subsection makefiles Makefiles
14
15-# Within an appropriate subdirectory of the \c doc directory,
16 create a directory for your documentation, e.g.
17 \code
18 cd doc/axisym_navier_stokes/
19 mkdir spin_up
20 \endcode
21-# Add this new directory to the parent directory's \c Makefile.am
22-# Add the entry \c doc/axisym_navier_stokes/spin_up to the file \c
23 config/configure. \c ac_scripts/doc.\c dir_list
24-# Create a new \c *.\c txt file with the same name as the
25 newly-created directory, e.g.
26 \code
27 touch spin_up.txt
28 \endcode
29 This is the "source" file from which the documentation will be
30 generated.
31-# From an existing documentation directory, copy across the following
32 files into the newly created directory:
33 - \c Makefile.am
34 - \c Doxyfile
35-# In \c Makefile.am after \c "docfile =", add the stem of the \c *.\c
36 txt file created in step 4, e.g.
37 \code
38 docfile = spin_up
39 \endcode
40-# In \c Doxyfile update the (relative) path to the demo-driver directory. Two
41 entries must be updated, one following \c "INPUT" and the other
42 following \c "EXAMPLE_PATH". To find these, search for
43 \c "../..".
44-# Return to \c oomph-lib's top-level directory and re-run
45 \c autogen.\c sh:
46 \code
47 ./autogen.sh
48 \endcode
49
50\subsection figures_setup Figures
51
52If your documentation is to contain figures or animations, the
53 following subdirectories must be created within your documentation
54 directory:
55 - \c figures
56 - This must contain all figures used in the documentation in both
57 \c *.\c gif and \c *.\c eps formats (\c *.\c gif is used for the \c html
58 documentation whilst \c *.\c eps is used for \c LaTeX).
59 - \c non_distfigures
60 - This contains any additional files (etc.) used to create or
61 maintain the documentation. It is good practise to always keep any
62 \c *.\c lay and \c *.\c lpk files that were used to create
63 figures, along with any macro (\c *.\c mcr) files.

67\section write Writing the documentation
68
69The "source" of the documentation is contained in the \c *.\c txt file (a
70mixture of \c html/doxygen markup). The title must follow the tag
71\c \\mainpage and be all on one line, e.g.
72\code
73\mainpage Demo problem: Spin-up of a viscous fluid
74\endcode
75The main body of the text just follows as in a \c LaTeX document, with
76line spacings indicating paragraph breaks.
77
78\subsection html HTML
79
80Any standard html tags can be used (for example, \c <hr> inserts a
81horizontal line). Hyperlinks are inserted in the following way:
82\code
83This is illustrated in an <A HREF="../figures/my_movie.avi">animation</A>.
84\endcode
85Note that the \c ".." is present in the path above because during the
86build process the documentation is created in subdirectories of the
87directory in which the source file is located. See the section \ref
88generate below for more details.
89
90To link to the documentation of another demo driver, \c
91poisson/one_d_poisson for example, hyperlink to the \c index.html file
92within the \c html directory of that demo driver's documentation. Note
93that because the processed (\c html) version of the documentation you are
94working on will live in its own \c html subdirectory, it is necessary
95to go up three directories in order to be in \c oomph-lib's \c doc directory.
96\code
97...see <A HREF="../../../poisson/one_d_poisson/html/index.html">the
98Poisson tutorial</A>.
99\endcode

101\subsection sections Sections
102
103Sections are created as follows:
104\code
105\section section_label This is the title of my section
106\endcode
107
108Likewise, subsections are created in the following way:
109\code
110\subsection subsection_label This is the title of my subsection
111\endcode
112
113Any \c LaTeX section types can be used in this way. To link to a
114section within a document, use the syntax \c \\ref \c section_label,
115as in this example:
116\code
117...can be found in the section \ref theory below.
118\endcode

120\subsection equations Equations
121
122Equations are generated as in \c LaTeX except \c \\f must be added
123before \c \$ or \c [, \c ].
124- E.g. inline maths:
125 \code
126 ...is given by \f$ \sin(x) \f$
127 \endcode
128- E.g. equation environment:
129 \code
130 \f[
131 \sin(x)
132 \f]
133 \endcode
134
135To label equations, the normal \c LaTeX system cannot be used. Instead,
136mark an equation with the tag \c \@E\c [LABEL\c ]\@ and refer to it later
137using \c \@R\c [LABEL\c ]\@, e.g.
138
139\image html labelling_example.gif " "
140\image latex labelling_example.eps " " width=0.71\textwidth
141
142and then later on...
143\code
144...is given by equation (@R[eqn:cont]@).
145\endcode

147\subsection lists Lists
148
149To create bullet point lists, precede each item with a \c -, e.g.
150\code
151- First item
152- Second item
153\endcode
154To create enumerated lists, precede each item with a \c -#, e.g.
155\code
156-# First item
157-# Second item
158\endcode

160\subsection figures Figures
161
162A figure with the filename \c my_figure.\c * is inserted in the
163following way:
164
165\image html insert_figures.gif " "
166\image latex insert_figures.eps " " width=0.71\textwidth
167
168Note the space between the last character in the caption and the
169quotation marks. Like the equation labelling, this line is processed
170by the \c txt2h.sh script (which is run automatically as part of the
171\c make process) and replaced with the necessary commands that tell
172\c doxygen
173to use the \c *.\c gif files for the html documentation and the \c *.\c
174eps files for the \c LaTeX documentation.
175
176\subsection code Code
177
178To insert single words of code into prose, precede the word with a
179\c \\c, e.g.
180\code
181The function \c FiniteElement::output(...) is used to...
182\endcode
183
184To include blocks of code such as the one immediately above this line
185of text, use the \c \\code environment, e.g.
186
187\image html code_environment.gif " "
188\image latex code_environment.eps " " width=0.71\textwidth
189
190To include sections of the demo code which you are documenting,
191e.g. the main function of \c spin_up.\c cc, use the following syntax:
192\code
193\dontinclude spin_up.cc
194\skipline start_of_main
195\until end of main
196\endcode
197This only works if \c start_of_main exists somewhere in \c spin_up.\c cc
198file, but any word(s) can be used as a start/endpoint. However, <B>do
199not use dashes as targets</B> because more recent versions of doxygen
200get very confused by this, so <b>don't</b> do
201\code
202\skipline ----
203\endcode
204say.

207\subsection misc Miscellaneous
208
209- To tell \c doxygen to ignore everything in the source file below a
210 certain point, denote this point with \c \@\@END\@\@.
211- To tell \c doxygen that a certain section of the source file is only
212 to be included in the \c html version of the documentation and
213 omitted in the \c LaTeX version, enclose this section within \c
214 \c \\htmlonly and \c \\endhtmlonly tags. CAREFUL: With recent
215 version of doxygen, this has caused problems with certain commands
216 not being interpreted correctly. Best not to use this... The
217 following item is a work-around:
218- Add the variable \c suppress_latex_in_this_directory to the
219 Makefile.am and set it to 1 to bypass the generation of latex-based
220 documentation for a specific directory (which may contain difficult
221 to render tables etc. and therefore cause latex to hang...). Here's
222 an example of a Makefile.am from the directory
223 \c doc/order_of_action_functions:
224 \include Makefile.am
225- To tell \c doxygen that a certain section of the source file is only
226 to be included in the \c LaTeX version of the documentation and
227 omitted in the \c html version, enclose this section within \c
228 \c \\latexonly and \c \\endlatexonly tags.

232\section generate Generating the documentation
233
234Once the source file has been written, simply type \c make in the
235documentation directory to build the \c html and \c LaTeX versions,
236e.g.
237\code
238cd doc/axisym_navier_stokes/spin_up
239make
240\endcode
241Two subdirectories, \c html and \c latex, are now created containing
242the two versions of the documentation. A \c *.\c pdf file of the
243\c LaTeX version is also placed in the current directory.
