3\mainpage Example problem: The spatially-adaptive solution of the azimuthally Fourier-decomposed 3D Helmholtz equation
4
5
6In this document we discuss the spatially-adaptive
7finite-element-based solution of the 3D
8Helmholtz equation in cylindrical polar coordinates, using a
9Fourier-decomposition of the solution in the azimuthal direction.
10
11The driver code is very similar to the one discussed in
12<a href="../../sphere_scattering/html/index.html">another tutorial
13</a> -- the main purpose of the current tutorial is to demonstrate
14the use of spatial adaptivity on unstructured meshes.

20\section scattering A specific example
21We will solve the azimuthally Fourier-decomposed Helmholtz equation
22\f[
23\nabla^2 {u_{N}}(r,z) + \left(k^2-\frac{N^2}{r^2}\right) u_N(r,z) = 0,
24 \ \ \ \ \ \ \ \ \ \ \ \ (1)
25\f]
26where \f$ N \f$ is the azimuthal wavenumber,
27in the finite domain \f$ 1 < \sqrt{r^2 + z^2} < 3 \f$.
28We impose the Sommerfeld radiation condition at the outer boundary
29of the computational domain at \f$ \sqrt{r^2 + z^2} = 3\f$,
30using a Dirichlet-to-Neumann mapping, and apply flux boundary condition
31on the surface of the unit-sphere (where \f$ \sqrt{r^2 + z^2} = 1 \f$)
32such that the exact solution is given by
33\f[
34u_N(r,z)=u_N^{[exact]}(r,z)=\sum_{l=N}^{N_{\rm terms}}
35h_{l}^{(1)}(k\sqrt{r^2+z^2}) \ P_{l}^{N}\left(\frac{z}{\sqrt{r^2+z^2}}\right).
36\f]
37This solution corresponds to the superposition of several outgoing
38waves that emerge from the unit sphere.
39
40The two plots below show a comparison between the exact and computed
41solutions for \f$ N_{\rm terms}=6 \f$ , a Fourier wavenumber
42of \f$ N=1 \f$ , and a (squared) Helmholtz wavenumber of \f$ k^2 = 10 \f$.

56\section num_soln The numerical solution

64 <CENTER>
65 <A HREF="../../../../demo_drivers/fourier_decomposed_helmholtz/sphere_scattering/sphere_scattering.cc">
66 demo_drivers/fourier_decomposed_helmholtz/sphere_scattering/sphere_scattering.cc

69and
70 <CENTER>
71 <A HREF="../../../../demo_drivers/fourier_decomposed_helmholtz/sphere_scattering/unstructured_sphere_scattering.cc">
72 demo_drivers/fourier_decomposed_helmholtz/sphere_scattering/unstructured_sphere_scattering.cc
73 </A>
74 </CENTER>
75shows the main differences required to discretise the computational
76domain with an adaptive, unstructured mesh:
77- The provision of the functions \c actions_before/after_adapt() to
78 detach/re-attach the \c FaceElements that are used to enforce
79 the Neumann boundary conditions before and after every spatial
80 adaptation, and to pass the physical parameters to the newly
81 created bulk elements.
82 \n\n
83- The generation of an unstructured mesh whose curvilinear boundaries are
84 represented by \c GeomObjects -- this ensures that the domain boundaries
85 become increasingly well resolved under mesh refinement.
86.
87That's all!
